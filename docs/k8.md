# Local Kubernetes Deployment + Stress Testing

### Tools you need

| Tool | Purpose | Install |
|------|---------|---------|
| **minikube** | Local single-node K8s cluster | `winget install minikube` |
| **kubectl** | K8s CLI | bundled with minikube |
| **k6** | Load / stress testing | `winget install k6` |
| **Docker Desktop** | Build images | already needed for compose |

---

### 1. Start the cluster

```powershell
minikube start --cpus=4 --memory=8192 --driver=docker
minikube addons enable metrics-server   # required for HPA CPU metrics
minikube addons enable ingress          # NGINX ingress for gateway + web
```

---

### 2. Point Docker at minikube's registry

```powershell
# All subsequent `docker build` calls push into minikube's image cache
& minikube -p minikube docker-env --shell powershell | Invoke-Expression
```

Build each image (run from repo root so the build context includes the whole monorepo):

```powershell
docker build -f apps/auth-service/Dockerfile  -t notes/auth-service:dev .
docker build -f apps/notes-service/Dockerfile -t notes/notes-service:dev .
docker build -f apps/gateway/Dockerfile       -t notes/gateway:dev .
docker build -f apps/web/Dockerfile           -t notes/web:dev .
```

---

### 3. Create K8s manifests

Create `infra/k8s/` with one file per concern. The structure:

```
infra/k8s/
  namespace.yaml
  auth-db.yaml          # StatefulSet + Service
  notes-db.yaml         # StatefulSet + Service
  auth-service.yaml     # Deployment + Service + HPA
  notes-service.yaml    # Deployment + Service + HPA
  gateway.yaml          # Deployment + Service + HPA
  web.yaml              # Deployment + Service
  ingress.yaml          # NGINX Ingress rules
  secrets.yaml          # RSA keys, DB passwords (gitignored)
  configmap.yaml        # Non-secret env vars
```

Key snippets for each layer:

**namespace.yaml**
```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: notes
```

**configmap.yaml** — all non-secret config
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: notes-config
  namespace: notes
data:
  NODE_ENV: "production"
  JWT_ISSUER: "notes-auth-service"
  JWT_AUDIENCE: "notes-api-gateway"
  CORS_ORIGIN: "http://notes.local"
  COOKIE_SECURE: "false"      # true behind TLS
  COOKIE_SAME_SITE: "lax"
  JWT_REFRESH_TTL_DAYS: "30"
  AUTH_SERVICE_URL: "auth-service.notes.svc.cluster.local:50051"
  NOTES_SERVICE_URL: "notes-service.notes.svc.cluster.local:50052"
```

**secrets.yaml** — base64-encode your RSA keys, DB URLs, etc.
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: notes-secrets
  namespace: notes
type: Opaque
stringData:
  AUTH_DB_URL: "postgresql://auth:secret@auth-db.notes.svc.cluster.local:5432/auth"
  NOTES_DB_URL: "postgresql://notes:secret@notes-db.notes.svc.cluster.local:5432/notes"
  JWT_PRIVATE_KEY: |
    -----BEGIN RSA PRIVATE KEY-----
    ...
  JWT_PUBLIC_KEY: |
    -----BEGIN RSA PUBLIC KEY-----
    ...
```

**auth-service.yaml** — Deployment + ClusterIP Service + HPA
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: auth-service
  namespace: notes
spec:
  replicas: 2
  selector:
    matchLabels:
      app: auth-service
  template:
    metadata:
      labels:
        app: auth-service
    spec:
      containers:
        - name: auth-service
          image: notes/auth-service:dev
          imagePullPolicy: Never          # use local minikube image
          ports:
            - containerPort: 50051
          envFrom:
            - configMapRef:
                name: notes-config
            - secretRef:
                name: notes-secrets
          resources:
            requests:
              cpu: "100m"
              memory: "128Mi"
            limits:
              cpu: "500m"
              memory: "512Mi"
          readinessProbe:
            grpc:
              port: 50051
            initialDelaySeconds: 5
            periodSeconds: 5
---
apiVersion: v1
kind: Service
metadata:
  name: auth-service
  namespace: notes
spec:
  selector:
    app: auth-service
  ports:
    - port: 50051
      targetPort: 50051
  clusterIP: None   # headless — gRPC client does its own load balancing
---
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: auth-service-hpa
  namespace: notes
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: auth-service
  minReplicas: 2
  maxReplicas: 10
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 60
```

Repeat the same Deployment + Service + HPA pattern for `notes-service` (port 50052) and `gateway` (port 3000, use a regular ClusterIP service instead of headless since it's HTTP).

**ingress.yaml**
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: notes-ingress
  namespace: notes
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
    - host: notes.local
      http:
        paths:
          - path: /api
            pathType: Prefix
            backend:
              service:
                name: gateway
                port:
                  number: 3000
          - path: /
            pathType: Prefix
            backend:
              service:
                name: web
                port:
                  number: 5173
```

Add `notes.local` to your hosts file:
```powershell
# Run PowerShell as Administrator
Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "127.0.0.1  notes.local"

# Keep this running while testing (separate terminal)
minikube tunnel
```

### 5. Stress test with k6

Create `tests/load/stress.js`:

```js
import http from "k6/http";
import { check, sleep } from "k6";

// Ramp: 0→50 VUs over 1 min, hold 3 min, ramp down 1 min
export const options = {
  stages: [
    { duration: "1m",  target: 50  },
    { duration: "3m",  target: 50  },
    { duration: "1m",  target: 100 },
    { duration: "3m",  target: 100 },
    { duration: "1m",  target: 0   },
  ],
  thresholds: {
    http_req_failed:   ["rate<0.01"],   // <1% errors
    http_req_duration: ["p(95)<500"],   // 95th percentile < 500ms
  },
};

const BASE = "http://notes.local/api";

export function setup() {
  const email = `load-${Date.now()}-${Math.floor(Math.random() * 1_000_000)}@test.com`;
  const signup = http.post(
    `${BASE}/auth/signup`,
    JSON.stringify({ email, password: "Password123!", deviceName: "k6" }),
    { headers: { "Content-Type": "application/json" } },
  );
  check(signup, { "signup 201": (r) => r.status === 201 });
  if (signup.status !== 201) return { token: null };

  const token = signup.json("data.tokens.accessToken");
  if (typeof token !== "string" || token.length === 0) return { token: null };

  return { token };
}

export default function (data) {
  const token = data?.token;
  if (typeof token !== "string" || token.length === 0) return;

  // create a note
  const create = http.post(
    `${BASE}/notes`,
    JSON.stringify({ title: "Load note", contentType: "TEXT", content: JSON.stringify({ text: "hello k6" }) }),
    { headers: { "Content-Type": "application/json", Authorization: `Bearer ${token}` } },
  );
  check(create, { "create 201": (r) => r.status === 201 });

  // list notes
  const list = http.get(`${BASE}/notes?page=1&pageSize=20`, {
    headers: { Authorization: `Bearer ${token}` },
  });
  check(list, { "list 200": (r) => r.status === 200 });

  sleep(1);
}
```

Run it:
```powershell
k6 run tests/load/stress.js
```

If k6 still shows `signup 201` as 0% with fast response times, check for gateway rate limiting (`429`) and reset limiter state:

```powershell
kubectl rollout restart deployment/gateway -n notes
kubectl rollout status deployment/gateway -n notes
```

### 6. Watch autoscaling in real time

Open two terminals side by side while k6 runs:

**Terminal A — pod count**
```powershell
kubectl get pods -n notes -w
# You'll see pods like:
# gateway-xxxx-1   Running
# gateway-xxxx-2   Running   ← spawned by HPA
# gateway-xxxx-3   Pending   ← spawning...
```

**Terminal B — HPA status**
```powershell
kubectl get hpa -n notes -w
# NAME               REFERENCE             TARGETS    MINPODS  MAXPODS  REPLICAS
# auth-service-hpa   Deployment/auth-svc   68%/60%    2        10       4
# gateway-hpa        Deployment/gateway    72%/60%    2        10       6
```

**Terminal C — live CPU/memory**
```powershell
kubectl top pods -n notes
```

You'll observe the replica count climbing as CPU crosses 60%, then shrinking back ~5 minutes after load drops (the HPA cool-down window is 5 min by default; tune it with `--horizontal-pod-autoscaler-downscale-stabilization` on the controller).

---

### Key things to understand

- **Scale-out trigger**: HPA polls metrics-server every 15 s. When average CPU across existing pods exceeds 60%, it calculates `desiredReplicas = ceil(currentReplicas × (currentCPU / 60))` and adds pods up to `maxReplicas: 10`.
- **Scale-in delay**: Kubernetes deliberately waits 5 minutes of sustained low usage before removing pods to avoid flapping.
- **gRPC load balancing**: gRPC over HTTP/2 is connection-multiplexed. With a headless Service, the gateway creates a separate sub-connection per pod, so traffic distributes. With a regular ClusterIP it would go through kube-proxy (L4), which doesn't rebalance in-flight streams — use headless for the gRPC services.
- **Database**: StatefulSets keep the DB pods stable across rescheduling. In production you'd use a managed DB (RDS, Cloud SQL) instead.
- **Persistent volumes**: Add `PersistentVolumeClaim` to each StatefulSet so data survives pod restarts in minikube (`storageClassName: standard`).

## Implementation status in this repository

Implemented assets:
- Kubernetes manifests under `infra/k8s`

To execute end-to-end locally:
1. Install minikube and k6 (kubectl already available through Docker Desktop in many setups).
